What is Adversary Emulation?

Adversary Emulation is a type of security testing where defenders mimic the known TTPs of a specific threat actor to verify how well their organization can detect and respond to those specific behaviors.

Unlike traditional red teaming, which is goal-oriented (e.g., "get the domain admin"), emulation is behavior-oriented.

Emulation vs. Simulation

While often used interchangeably, there is a subtle difference in a CTI context:

  • Simulation: Creating a custom or synthetic "threat-like" behavior to test a control.
  • Emulation: Following a specific "recipe" or playbook derived directly from CTI reports about a real-world actor (e.g., "Emulate APT29's use of ISO files for initial access").

Tools and Frameworks

Adversary emulation is often performed using specialized tools mapped to the MITRE ATT&CK framework:

  • Atomic Red Team: A library of simple, modular tests that every security team can execute to test their defenses.
  • CALDERA: An automated adversary emulation system that allows for the creation and execution of complex operation chains.
  • MITRE Emulation Plans: Publicly available, step-by-step guides for emulating sophisticated actors like FIN6 or APT29.

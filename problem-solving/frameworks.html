Structuring Intelligence

Analyzing modern cyber threat campaigns requires structure. Threat intelligence frameworks provide a standardized taxonomy and model to dissect, catalog, and compare threat actor behavior.

Using frameworks allows analysts to avoid unstructured assumptions, organize massive volumes of indicators, and communicate threats clearly to both technical responders and executive leadership.

The Lockheed Martin Cyber Kill Chain

The Cyber Kill Chain is a linear model that represents the sequential phases of a cyberattack, helping defenders identify opportunities to disrupt the adversary.

  • 1. Reconnaissance: Gathering target intelligence (e.g., email addresses, open ports).
  • 2. Weaponization: Coupling malicious payloads with exploits (e.g., creating a malicious document).
  • 3. Delivery: Transmitting the payload (e.g., via spear-phishing email or USB).
  • 4. Exploitation: Triggering the malicious code to exploit a vulnerability.
  • 5. Installation: Establishing a backdoor or persistent presence on the asset.
  • 6. Command & Control (C2): Setting up a channel for remote administration.
  • 7. Actions on Objectives: Executing target actions (e.g., exfiltrating data, deploying ransomware).

The Diamond Model

The Diamond Model analyzes a single intrusion event by mapping four interconnected vertices that establish the relationship between actor and victim.

  • Adversary: The actor or campaign group responsible (e.g., APT29, threat group, insider).
  • Capability: The tools, malware, exploits, and specific skills used in the event.
  • Infrastructure: The IP addresses, domain names, C2 servers, or email accounts utilized.
  • Victim: The target organization, country, sector, or specific individual assets.
  • Axes & Features: Combines a Technical Axis (Capability ↔ Infrastructure) and Social-Political Axis (Adversary ↔ Victim) with metadata like timestamp, phase, and result.

MITRE ATT&CK

ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) is a comprehensive, globally-accessible matrix of threat actor behaviors observed in the wild.

  • Tactics: The adversary's immediate goals (the "Why", e.g., Persistence, Privilege Escalation).
  • Techniques: The methods used to achieve a tactic (the "How", e.g., Registry Run Keys).
  • Sub-techniques: More granular descriptions of the technique (e.g., User Init Mpr Logon Script).
  • Procedures: The specific implementation of a technique by a group (e.g., APT28 using a custom PowerShell script).

Synthesizing Frameworks

Modern CTI analysts do not use frameworks in isolation. Instead, they combine them to achieve a holistic view of threat campaigns.

  • The Kill Chain provides the timeline and progression of an intrusion over time.
  • The Diamond Model establishes attribution, infrastructure pivots, and strategic victimology.
  • MITRE ATT&CK provides the specific technical blueprints and detection engineering mappings.
  • Synthesis: Mapping Diamond Model events along Kill Chain phases, and defining each phase's capabilities using MITRE ATT&CK technique IDs.

Introduction to Threat Indicators

An Indicator of Compromise (IoC) is an artifact observed on a network or host that, with a high degree of confidence, points to an active or past system intrusion.

Indicators are the operational building blocks of threat detection. However, they have a limited shelf life (decay rate) and vary greatly in their utility and the effort required for an adversary to modify them.

The Pyramid of Pain

Designed by security researcher David Bianco, the Pyramid of Pain rates indicators based on the level of friction (or "pain") an organization inflicts on an adversary by detecting and blocking them.

  • Hash Values (Bottom): Trivial for actors to change. Inflicts zero pain.
  • IP Addresses & Domains: Easy/Simple to change via proxy or registration.
  • Network & Host Artifacts: Annoying to change (e.g., custom protocols, file names).
  • Tools: Challenging. Requires the actor to find or compile new utilities.
  • TTPs (Top): Extremely painful. Requires changes to training, playbooks, and habits.

Low-Level Indicators (Trivial & Easy)

These indicators are easy to collect and deploy, but represent the lowest utility because adversaries can automate their evasion.

  • File Hashes: MD5, SHA-256 values of files. Changing a single byte in a payload changes its hash completely.
  • IP Addresses: Temporary C2 addresses. Threat actors route traffic through VPNs, Tor, or compromised consumer routers.
  • Domain Names: Domain generation algorithms (DGAs) allow actors to dynamically register hundreds of domains daily.

Mid-Level Indicators (Host & Network Artifacts)

Detecting mid-level indicators targets the footprint left behind by the adversary's malware configurations and administrative utilities.

  • Network Artifacts: Hardcoded User-Agent strings, custom HTTP headers, SSL/TLS JA3/JA4 fingerprints, or unusual SMB port communication.
  • Host Artifacts: Specific registry paths created for persistence, temporary file paths (e.g., C:\Windows\Temp\), or unique mutex names.
  • Utility: Restricts the actor's use of specific configurations, forcing them to recompile payloads or alter infrastructure scripts.

High-Level Indicators (TTPs & Behaviors)

Behavioral indicators focus on how the attack is conducted (Tactics, Techniques, and Procedures) rather than the technical values used.

  • Behavioral Detection: Identifying execution patterns, such as Microsoft Word launching cmd.exe which subsequently invokes powershell.exe.
  • EDR/Sigma Rules: Writing logic rules (e.g., Sigma, YARA) that flag broad adversarial behaviors (like LSASS memory dumping).
  • Resilience: Because behavioral profiles reflect core operational habits, they are highly resilient and difficult for actors to bypass without redesigning their attack path.

Managing Indicators at Scale

To prevent performance degradation of security tools, CTI operations must manage indicators systematically throughout their lifecycle:

  • Deduplication & Enrichment: Merging redundant indicator records and adding context (e.g., linking an IP to a known actor profile).
  • False Positive Mitigation: Whitelisting legitimate cloud domains, corporate IP ranges, and common OS system files.
  • Indicator Aging & Decay: Automatically expiring or lowering the confidence scores of indicators over time (e.g., active C2 IPs decay faster than malware hashes).

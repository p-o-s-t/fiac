Defining Indicators (IoCs)

An Indicator of Compromise (IoC) is an artifact observed on a network or in an operating system that, with high confidence, indicates a computer intrusion.

Not all indicators are created equal. We categorize them by their stability, complexity, and how easily an adversary can change them.

1. Atomic Indicators

Atomic indicators are those which cannot be broken down into smaller parts and retain their meaning in the context of an intrusion.

  • IP Addresses: e.g., 185.25.12.34
  • Domain Names: e.g., evil-malware-c2.com
  • Email Addresses: e.g., phish-king@gmail.com

Adversary effort: Low. These are very easy for an attacker to change.

2. Computed Indicators

Computed indicators are those which are derived from data, often involving a mathematical function.

  • File Hashes: SHA-256, MD5, or SHA-1 hashes of malicious files.
  • JA3/JA4 Fingerprints: Unique identifiers for SSL/TLS handshakes.
  • Fuzzy Hashes: e.g., SSDEEP, which can identify similar (but not identical) files.

Adversary effort: Medium. Changing a single bit in a file changes the hash, but changing the TLS implementation takes more work.

3. Behavioral Indicators

Behavioral indicators represent the how of an attack. They describe the patterns of activity that an adversary performs.

  • Execution Patterns: cmd.exe launching powershell.exe with an encoded command.
  • Network Patterns: A server making unusual DNS requests for random subdomains (DNS Tunneling).
  • Registry Keys: Specific keys created for persistence after a reboot.

Adversary effort: High. This is David Bianco's "Tough" or "Painful" level on the Pyramid of Pain.

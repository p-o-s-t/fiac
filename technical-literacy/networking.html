Network Architecture & The OSI Model

The OSI (Open Systems Interconnection) model breaks network communication into seven logical layers, defining how data is encapsulated and transmitted across systems:

  • Encapsulation: As data moves down the stack, each layer wraps it in a new header (e.g., Layer 4 adds TCP headers, Layer 3 adds IP headers).
  • Analysis Alignment: Threat analysis requires matching attack vectors to their layer (e.g., SQLi at Layer 7, IP spoofing at Layer 3, ARP poisoning at Layer 2).
  • Layers 1-7: Physical, Data Link, Network, Transport, Session, Presentation, Application.

Transport Protocols: TCP, UDP, & Handshakes

At Layer 4, transport protocols determine how packets are established and validated between endpoints:

  • TCP Handshake: Connection-oriented. Uses a **Three-Way Handshake** (SYN → SYN-ACK → ACK) to sync sequence numbers. Essential for stateful firewalls.
  • TCP Flags: Packets contain control flags (SYN, ACK, RST, FIN, PSH, URG). An abundance of RST or SYN packets often indicates active port scanning.
  • UDP Protocol: Connectionless and stateless. Speed-focused with no recovery checks, making it ideal for DNS and DHCP but also abused for amplification DDoS.

Application Protocols & Common Ports

Services use designated logical ports to route traffic. CTI analysts inspect port traffic to locate command-and-control (C2) channels:

  • 22 (SSH) & 3389 (RDP): Admin ports heavily targeted for initial access, credential brute-forcing, and lateral movement.
  • 80 (HTTP) & 443 (HTTPS): Web protocols. Malware frequently routes C2 traffic over port 443 to blend in with legitimate corporate web browsing.
  • 445 (SMB): Server Message Block, used for network file sharing. Famous for lateral propagation (e.g., WannaCry using EternalBlue exploit).

DNS (Domain Name System) in CTI

DNS maps human-readable domains to numeric IP addresses. It is a critical data source for tracking infrastructure:

  • DGA (Domain Generation Algorithms): Malware dynamically generates hundreds of random domains daily to bypass static blocklists.
  • Fast Flux: Rapidly changing the IP addresses pointed to by a single C2 domain to evade hosting-level takedowns.
  • Passive DNS: Historic databases recording historical IP mappings of domains, allowing analysts to uncover related threat domains.

Network Traffic Analysis

Analysts inspect network flows and packets to investigate active threat campaigns:

  • PCAP (Packet Capture): Full packet recordings showing headers and payloads. Essential for deep analysis but storage-intensive.
  • NetFlow: Metadata showing source/destination IPs, ports, timestamps, and byte sizes. Highly efficient for broad anomaly detection.
  • TLS Metadata: Even when payload data is encrypted, certificate fields (e.g., issuer, serial number) reveal infrastructure characteristics.

The OSI Model

The OSI (Open Systems Interconnection) model breaks network communication into seven layers. Understanding these is essential for locating where an attack is occurring:

  • 7. Application: Interface for users/apps (HTTP, FTP).
  • 6. Presentation: Data formatting and encryption (SSL/TLS).
  • 5. Session: Managing connections (NetBIOS).
  • 4. Transport: End-to-end communication (TCP, UDP).
  • 3. Network: Logical addressing and routing (IP, ICMP).
  • 2. Data Link: Physical addressing (MAC addresses, ARP).
  • 1. Physical: The actual hardware (cables, hubs).

TCP vs UDP

At the Transport Layer, two protocols dominate how data is moved:

  • TCP (Transmission Control Protocol): Connection-oriented, ensuring reliable data delivery. Used by HTTP, SSH, and SMTP. It uses a three-way handshake (SYN, SYN-ACK, ACK).
  • UDP (User Datagram Protocol): Connectionless and fast, with no guarantee of delivery. Used by DNS, DHCP, and media streaming.

CTI analysts monitor for unusual volumes of either protocol to identify scanning or exfiltration.

Common Ports

Network services use standard ports. Analysts look for communication on these ports to identify active services:

  • 22 (SSH): Secure Shell, used for remote access.
  • 80 (HTTP): Web traffic, often used by malware for C2 (Command and Control).
  • 443 (HTTPS): Encrypted web traffic, harder to inspect.
  • 3389 (RDP): Remote Desktop Protocol, a common target for brute-force and initial access.

DNS and CTI

The Domain Name System (DNS) translates human-readable domain names (example.com) into IP addresses (93.184.216.34).

DNS is a goldmine for CTI:

  • DGA (Domain Generation Algorithms): Used by malware to generate thousands of domains for C2.
  • Fast Flux: A technique where multiple IP addresses are assigned to a single domain name, which change rapidly.
  • Passive DNS: Historic records showing which IP a domain pointed to in the past, helping track actor infrastructure.

Network Boundary Security

Boundary defense platforms inspect and filter traffic at the perimeter of the corporate network to block ingress attacks and flag egress anomalies:

  • NGFW (Next-Generation Firewalls): Filter traffic based on ports, protocols, and application awareness (e.g., detecting TOR protocols hidden over port 443).
  • WAF (Web Application Firewalls): Positioned in front of web applications to detect and block web-based exploits (like SQL Injection, Cross-Site Scripting, and path traversal).
  • IDS/IPS (Intrusion Detection/Prevention): Signature and behavioral-based engines that flag known exploit patterns or block traffic dynamically.

Endpoint Protection Systems

Endpoint security platforms defend user workstations, active servers, and mobile devices against local execution of malware:

  • Legacy Antivirus (AV): Relies on static cryptographic hashes to block known files. Easily bypassed by changing a few bytes of code.
  • EDR (Endpoint Detection & Response): deploys lightweight agents that monitor process executions, file system writes, and memory calls (detecting threats like process hollowing).
  • XDR (Extended Detection & Response): Integrates endpoint EDR logs with network firewalls, email filters, and cloud telemetry for cross-domain investigations.

Log Aggregation & SIEM

Security Information and Event Management (**SIEM**) systems act as the primary operational brain for the Security Operations Center (SOC):

  • Normalization: Converting log records from firewalls (JSON), Windows events (XML), and Linux system logs (syslog) into a standard schema.
  • Correlation Engines: Running real-time queries to find patterns of threat events across different systems (e.g., flagging a user logging in from two countries within an hour).
  • Forensics Storage: Storing historical audit trails to enable post-incident scoping and tracking of threat activity timelines.

Deception Technologies

Deception systems shift the advantage to the defender by deploying fake assets across the network to detect and monitor adversaries:

  • Honeypots: Decoy servers configured to look like high-value databases or domain controllers, generating high-fidelity alerts if accessed.
  • Canary Tokens: Hidden files, database entries, or API credentials scattered across workstations. If opened or used, they instantly alert security operations.

Vulnerability & Application Security

Proactive security requires identifying and remediation of system vulnerabilities before they are leveraged by threat actors:

  • Vulnerability Scanners (Nessus, Qualys): Automated platforms that probe host configurations to identify unpatched software and weak protocol settings.
  • Static/Dynamic Application Security Testing (SAST/DAST): Inspecting custom application source code (SAST) or running app binaries (DAST) to find logical security vulnerabilities.
  • SBOM (Software Bill of Materials): Inventory of all third-party libraries used in software, allowing teams to quickly locate systems vulnerable to newly disclosed CVEs (like Log4j).

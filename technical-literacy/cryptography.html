Encryption Types

Cryptography is the science of securing communication. CTI analysts must understand two main types of encryption:

  • Symmetric Encryption: Uses a single key for both encryption and decryption. Fast but key management is challenging. Examples: AES, ChaCha20.
  • Asymmetric Encryption: Uses a pair of keys: a public key (for encryption) and a private key (for decryption). Used for secure key exchange and digital signatures. Examples: RSA, ECC.

Hashing

A hash function takes an input of any size and produces a fixed-size string of characters. It is one-way and used for Integrity checks:

  • Malware Identification: Unique hashes (MD5, SHA-256) act as fingerprints for specific files.
  • Password Security: Storing hashes instead of plaintext passwords.
  • Integrity: Verifying that a downloaded file hasn't been tampered with.

SSL/TLS and Fingerprinting

SSL/TLS encrypts network traffic (HTTPS). Analysts use it for attribution:

  • Certificates: Information about the certificate issuer can link infrastructure to known actors.
  • JA4 Fingerprinting: A modern method for identifying client/server software based on how they negotiate a TLS connection. This helps identify malware and C2 servers even when the traffic itself is encrypted.

Encryption in Malware

Threat actors leverage cryptography in various ways:

  • Ransomware: Encrypts victim data to extort payment, often using a combination of symmetric and asymmetric encryption.
  • C2 Communications: Malware encrypts traffic to its Command and Control server to bypass network security monitoring.
  • String Obfuscation: Malware authors encrypt strings (like C2 domains or registry keys) within the binary to hinder analysis.

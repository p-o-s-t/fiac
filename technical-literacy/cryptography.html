Cryptographic Encryption Types

Cryptography secures communication through mathematically complex key operations. CTI analysts dissect two main forms of encryption:

  • Symmetric Encryption: Uses a single secret key to both encrypt and decrypt data. Extremely fast but requires secure out-of-band key sharing. Algorithms: AES (Advanced Encryption Standard), ChaCha20.
  • Asymmetric Encryption: Uses a mathematically linked key pair: a *Public Key* (freely shared, encrypts data) and a *Private Key* (kept secret, decrypts data). Algorithms: RSA, ECC (Elliptic Curve Cryptography). Often used for key exchange protocols (Diffie-Hellman).

Cryptographic Hashing for CTI

Hash functions convert an input of arbitrary size into a unique, fixed-size mathematical representation. Hashes are strictly one-way and collision-resistant:

  • Payload Identification: Using MD5, SHA-1, and SHA-256 to create unique signatures (fingerprints) for malicious files.
  • Fuzzy Hashing: Using algorithms like **SSDEEP** or **TLSH** to measure file similarity, helping analysts identify modified variants of known malware.
  • Import Hashing (Imphash): Hashing the list of API functions imported by a Portable Executable. If two files have the same Imphash, they likely share the same source code structure, indicating a common origin.

SSL/TLS & CTI Fingerprinting

The **SSL/TLS** protocol encrypts network communications (HTTPS). While encryption hides payload data, the handshake metadata is a goldmine for tracking infrastructure:

  • Certificate Pivoting: Analysts pivot on TLS certificate fields (e.g., Issuer, Serial Number, Subject Alternative Names) to identify other C2 servers registered by the same threat group.
  • JA3/JA4 Fingerprinting: A method to identify the client application negotiating the TLS handshake based on cipher suites and extensions. Allows analysts to identify specific malware communication channels even if they are fully encrypted.

Encryption in Ransomware

Modern ransomware uses a sophisticated **hybrid encryption scheme** to rapidly lock data and prevent decryption without paying the ransom:

  • The Scheme: The ransomware generates a unique *Symmetric Key* (e.g., AES) local to the host to encrypt files quickly. It then encrypts this symmetric key using the attacker's embedded *Asymmetric Public Key*.
  • Decryption Control: The symmetric key cannot be decrypted without the attacker's matching *Private Key*, which never leaves the attacker's command server.

C2 Encryption & Obfuscation

Adversaries implement cryptographic mechanisms inside their payloads to evade perimeter security detection:

  • C2 Traffic Encryption: Malware encrypts HTTP payloads (using custom RC4 or AES) before transmission to ensure network IDS cannot inspect command strings.
  • String Encryption: Malware authors encrypt API strings and domain configurations inside the binary. The malware decrypts them in memory only when needed, bypassing static string analysis.

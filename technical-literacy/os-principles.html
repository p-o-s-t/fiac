Windows Internals & CTI

The Windows Operating System is the primary target for desktop malware. Understanding its internal architecture is key to finding adversary traces:

  • Registry Database: A hierarchical configuration database. Actors use it for persistence (e.g., Run keys, Services creation, Active Setup hijacking).
  • NTFS alternate Data Streams (ADS): A filesystem capability allowing data to be attached to an existing file descriptor, often abused to hide payloads.
  • Portable Executable (PE): The standard format for Windows executables, DLLs, and drivers. Analysts inspect PE headers (e.g., sections like `.text`, `.data`) to analyze malicious imports.

Linux Security & Logs

Linux is the foundation of the cloud and enterprise servers, making Linux-based attacks highly impactful:

  • Standard Logging: System logs (stored in `/var/log/`, like `secure`, `syslog`, `cron`) track user logins, cron executions, and service failures.
  • Executable & Linkable Format (ELF): The standard binary executable format for Linux, often targeted by cross-platform actors using languages like Go and Rust.
  • Virtual Filesystem (/proc): A pseudo-filesystem tracking active kernel memory and process executions. Checking `/proc/[PID]/exe` helps identify hidden run paths.

macOS Security Architecture

Built on the BSD Unix-based Darwin kernel, macOS deploys unique structural security controls:

  • TCC (Transparency, Consent, Control): The framework that manages application permissions to hardware (mic, camera) and files (Desktop, Documents).
  • System Integrity Protection (SIP): A kernel-level constraint preventing even the root user from modifying protected system folders.
  • Gatekeeper & XProtect: Apple's built-in controls. Gatekeeper verifies app cryptographic signatures, while XProtect provides signature-based malware detection.

OS Memory & Processes

Advanced adversaries manipulate memory structures directly to execute code without writing payloads to the disk:

  • Process Injection: Injecting malicious code into legitimate running processes (e.g., `explorer.exe`) via techniques like DLL Injection or Process Hollowing.
  • LSASS Dumping: Dumping the Local Security Authority Subsystem Service (LSASS) memory to harvest plaintext credentials and NTLM password hashes.
  • Virtual Memory: Understanding how the OS maps physical RAM to process address spaces (Heap for dynamic variables, Stack for execution tracking).

Virtualization & Containerization

Modern server environments run multiple logical operating systems on shared physical hardware:

  • Hypervisors (Type 1 & 2): Software that creates and runs Virtual Machines (VMs), providing strong hardware-level isolation.
  • Containers (Docker/Kubernetes): OS-level virtualization sharing the host OS kernel. Containers are faster than VMs but share resources, making them vulnerable to **Container Breakout** attacks where an attacker escapes to the host system.

Windows Operating System

The Windows OS is characterized by several key architectural components critical for CTI analysts:

  • NTFS (New Technology File System): Supports features like permissions, encryption, and Alternate Data Streams (ADS), which malware can use for hiding.
  • Registry: A hierarchical database storing configuration settings. Often used by malware for persistence (e.g., Run keys).
  • PE (Portable Executable) Files: The standard format for Windows executables, DLLs, and drivers. Understanding its headers and sections is vital for malware analysis.

Linux Operating System

Linux is a Unix-like OS widely used in servers and embedded devices:

  • Filesystem Hierarchy: Follows a standard structure (e.g., /etc for config, /var/log for logs, /tmp for temporary files).
  • Permissions: Uses a robust model (read, write, execute) for owners, groups, and others. Root is the all-powerful superuser.
  • ELF (Executable and Linkable Format): The standard binary format for Linux. CTI analysts look for ELF-based malware targeting cloud environments.

macOS

Built on a Unix base (Darwin), macOS includes unique security features:

  • Unix Foundation: Shares many command-line tools and filesystem similarities with Linux.
  • Security Features: Includes Gatekeeper (verifying apps), System Integrity Protection (SIP) (protecting system files), and XProtect (built-in malware scanning).
  • App Sandboxing: Restricts what data an app can access, limiting the potential impact of a compromise.

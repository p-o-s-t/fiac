Activity Group Clustering

Tracking involves grouping related malicious events into Activity Groups. Analysts look for shared characteristics across different campaigns to identify a single actor.

By clustering data, we can move from seeing isolated incidents to understanding a persistent threat actor's lifecycle.

Diamond Model Threading

Threading is the process of connecting multiple Diamond Models to show how an adversary's activity evolves over time.

  • Activity Threads: Linking events where the same infrastructure or capability is used against different victims.
  • Pivoting: Using an artifact from one vertex (e.g., a malware sample) to discover a new vertex (e.g., a C2 server).

The Rule of Two

When clustering activity, the Rule of Two is a common analytical standard:

An observation is considered part of a specific activity cluster if it shares at least two unique, high-confidence artifacts or TTPs with that cluster.

  • Example: Using the same custom backdoor AND the same unique C2 registration email.
  • This reduces the risk of misattribution based on a single, easily spoofed indicator.

Attribution Confidence Levels

Assigning responsibility for an attack requires careful, standardized evaluation:

  • Estimative Language: Expressing attribution claims using structured confidence terms (e.g., "highly likely," "moderate confidence") per ICD 209 guidelines.
  • False Flag Analysis: Identifying if an actor intentionally left misleading artifacts (like Russian language strings in a Chinese malware compile) to deceive analysts.
  • Temporal Correlation: Analyzing compile times and C2 activity logs to determine the operator's working hours and timezone.
  • Infrastructure Spoofing: Accounting for rented, shared, or compromised infrastructure (like VPN endpoints) which lowers direct attribution confidence.

Campaign Tracking Databases

CTI teams rely on specialized software and rulesets to map and track adversary footprints:

  • Graph Databases: Using systems like Neo4j or Maltego to map connections between domains, IP addresses, hashes, and target sectors.
  • Yara Rule Deployment: Deploying rules to retroactively scan malware repositories (e.g., VirusTotal) to find older, undocumented campaign samples.
  • TIP Correlation: Utilizing Threat Intelligence Platforms (e.g., MISP) to auto-correlate indicators across separate internal incidents.
  • Indicator Lifecycle Management: Dynamically retiring or aging out indicators that have been abandoned by the actor to reduce false positives.

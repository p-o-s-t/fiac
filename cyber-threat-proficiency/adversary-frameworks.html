Exploit Frameworks

Exploit Frameworks are used to deliver payloads and exploit vulnerabilities during the initial stages of an attack. Common examples include:

  • Metasploit: A widely used open-source framework with thousands of exploits and payloads for various platforms.
  • Core Impact: A commercial-grade penetration testing tool that automates complex exploitation tasks.

Post-Exploitation Frameworks

Post-Exploitation (C2) Frameworks are used by attackers to maintain persistence, move laterally, and exfiltrate data after initial access. Popular tools include:

  • Cobalt Strike: A powerful commercial tool favored by both red teams and advanced persistent threat (APT) groups.
  • Sliver: A modern, open-source cross-platform C2 framework developed by Bishop Fox.
  • Brute Ratel: A specialized C2 tool designed to evade modern EDR and antivirus solutions.

The Analyst's Role

CTI analysts must understand the artifacts left behind by these frameworks to detect and respond to threats effectively:

  • Network Artifacts: Identifying unique beaconing patterns, C2 communication protocols, and default certificates.
  • Host Artifacts: Detecting specific file names, registry keys, and process behavior associated with these tools.
  • Memory Forensics: Analyzing memory dumps to find framework-specific signatures and injected code.
